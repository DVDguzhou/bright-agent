# BrightAgent 商业模式说明

> 用于 `pitch deck` 的 `Business model / Unit economics` 页面，以及投资人追问时的补充口径

## 1. 早期商业模式应该怎么讲

BrightAgent 早期不适合把商业模式讲得太宽。最合理的说法是：

> 我们先验证“关键决策经验 Agent”在轻咨询场景中的付费闭环，再逐步扩展到订阅、供给侧工具和更标准化的 License 模式。

也就是说，先讲清 `C 端轻决策咨询`，再保留远期扩展。

## 2. 早期收入结构

### 2.1 按次付费

适用于单个高意图问题：

- 如 `19-49 RMB` 一次
- 用户购买的是一次完整问题链路的咨询权，而不是单条回复
- 适合校招选择、考研 vs 实习、转行判断等场景

### 2.2 套餐 / 月度问题包

适用于需要连续追问的用户：

- 如 `99-199 RMB / 月`
- 提供固定问题额度或对话轮次
- 适合一个阶段内持续决策，如找实习、准备转行、毕业规划

### 2.3 平台抽成

平台对每笔成交收取服务费：

- 早期建议 `15%-25%`
- 该比例足以覆盖支付、模型、审核、客服、纠纷处理和推荐分发的成本

## 3. 为什么用户愿意付费

用户不是为“AI”付费，而是为以下组合付费：

- 更贴近自己背景的建议
- 可持续追问
- 可匿名表达敏感问题
- 比真人咨询更低的价格和更快的响应

因此 BrightAgent 的本质不是工具订阅，而是低客单、高频次的经验服务消费。

## 4. 为什么供给者愿意参与

供给者获得的不是一次性打赏，而是可持续资产化收入：

- 过去只能发帖、做内容、偶尔私信回复
- 现在可把经验封装为一个持续接单的 Agent
- 不需要一直在线，也能扩大经验变现效率

对优质供给者来说，平台的价值是：

- 带来新的需求
- 降低履约成本
- 沉淀个人品牌与评分
- 提供结算、争议处理和身份背书

## 5. 平台的成本结构

投资人通常会问：AI 项目能不能有毛利。建议用下面的逻辑回答。

### 主要成本

- 基础模型调用成本
- 支付和退款成本
- 审核与客服成本
- 增长和供给冷启动成本

### 为什么毛利有机会成立

- 单次咨询的价值不由 token 数量决定，而由决策价值决定
- 底层模型可替换、可优化，模型成本中长期会下降
- 平台真正需要沉淀的是匹配、信任和供给管理能力，而不是自研底模
- 一旦用户产生复购，单次获客成本将被多次消费摊薄

## 6. 一个可讲得通的早期单元经济模型

下面是一套用于 `deck` 的示意模型，不要写得过满：

### 示例 A：按次咨询

- 用户支付：`29 RMB`
- 平台抽成：`20%`
- 平台收入：`5.8 RMB`
- 模型与支付等直接成本：`1-2 RMB`
- 单笔贡献毛利：`3.8-4.8 RMB`

### 示例 B：月度问题包

- 用户支付：`129 RMB`
- 平台抽成和服务收入：按结构化分成计入
- 若用户在一个月内形成多次咨询，平台收入和复购黏性都更稳定

### 投资人结论

BrightAgent 的关键不是把单笔利润做到极高，而是验证：

- 用户是否愿意为更贴近处境的建议付费
- 是否会在一个决策周期内连续复购
- 供给者是否愿意持续维护并提升回答质量

## 7. 为什么不要过早主打 License

`License` 是平台长期很有想象力的方向，但不是早期最容易被理解的切口。

原因：

- 早期投资人更关心真实用户是不是愿意付费
- `License` 更像平台化或企业化能力，需要在用户和供给基础之上成立
- 如果一开始就把核心收入写成 `License`，会让 deck 看起来离真实用户更远

因此建议对外口径是：

- `现在`：按次咨询 + 套餐 + 平台抽成
- `之后`：高质量标准化 Agent 可扩展为 License 或团购采购

## 8. 投资人最可能追问的 5 个问题

### 1. 用户为什么不直接问 ChatGPT？

因为用户买的不是“答案”，而是“与我背景相近的人在走过这条路后的判断”，这需要真实经验、上下文和持续追问。

### 2. 为什么不直接找真人前辈？

真人咨询成本高、响应慢、情绪负担大，而且很多敏感问题不方便反复问。经验 Agent 降低了时间、价格和表达门槛。

### 3. 供给质量怎么保证？

靠身份认证、经历标签、用户评分、对话回执和供给侧运营，而不是完全依赖模型回答本身。

### 4. 这个模式为什么不是很容易被复制？

产品界面容易复制，但真实供给、结构化经验、连续对话数据、评分体系和交易信任层会形成复利壁垒。

### 5. 退出怎么考虑？

不建议在主 deck 展开。标准回答是：先按能独立做大的公司去建设；如果未来形成有价值的供需网络和交易基础设施，自然会具备战略并购价值。

## 9. deck 中可直接使用的精简版

> BrightAgent 早期采用按次咨询、月度问题包和平台抽成的轻量商业模式。我们优先验证的是用户在关键决策场景中的付费意愿和复购行为，而不是过早铺开复杂的平台收费。随着高质量经验 Agent 的积累，平台再扩展至订阅、团购采购和 License 授权。

## 10. 你现在最该补的真实数字

下一版 deck 最值得补充的不是三年收入预测，而是下面这些真实数字：

- 单次咨询意向价格
- 首次付费转化率
- 一周 / 一月复购率
- 平均单次对话轮数
- 优质供给者留存率
- 退款率与纠纷率

这几个数字一旦有了，BrightAgent 的商业模式可信度会比任何宏大叙事都更强。
