# 阿里云 ECS 部署步步指南

适合 2核4G 服务器，从头到尾一步步操作。

---

## 第一步：购买 / 准备阿里云 ECS

1. 登录 [阿里云官网](https://www.aliyun.com) → 进入「云服务器 ECS」
2. 点击「创建实例」或「立即购买」
3. 配置建议：
   - **地域**：选离你用户近的（如华北、华东）
   - **规格**：2核 4GB 内存（轻量应用服务器也行）
   - **镜像**：**Ubuntu 22.04**
   - **系统盘**：40GB
   - **公网带宽**：按量付费 1M 或包年包月
4. 设置 root 密码（务必记住）
5. 购买完成，记下 **公网 IP**

---

## 第二步：开放安全组端口

1. 阿里云控制台 → ECS → 实例列表
2. 点击你的实例 → 左侧「安全组」→ 点击安全组 ID
3. 点击「配置规则」→「手动添加」
4. 添加以下规则（入方向）：

| 端口 | 授权对象 | 说明     |
|------|----------|----------|
| 22   | 0.0.0.0/0 | SSH 登录 |
| 80   | 0.0.0.0/0 | HTTP     |
| 443  | 0.0.0.0/0 | HTTPS    |

---

## 第三步：用 SSH 登录服务器

在**本地电脑**打开终端（Windows 用 PowerShell 或 CMD）：

```bash
ssh root@你的公网IP
```

提示输入密码时，输入购买时设置的 root 密码。登录成功会看到类似 `root@iZxxx:~#`。

---

## 第四步：安装 Docker

在**服务器**上依次执行：

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | sh

# 2. 启动 Docker 并设置开机自启
systemctl start docker
systemctl enable docker

# 3. 验证安装（应显示版本号）
docker --version
```

---

## 第五步：安装 Docker Compose

```bash
# 安装 Compose 插件
apt update
apt install docker-compose-plugin -y

# 验证
docker compose version
```

## 第六步：把项目上传到服务器

### 方式 A：用 Git（推荐）

```bash
# 先安装 git
apt install git -y

# 进入目录并克隆（替换为你的仓库地址）
cd /root
git clone https://github.com/你的用户名/regr.git regr
cd regr
```

### 方式 B：用 SCP 上传

在**本地电脑**（不是服务器）执行，把项目目录上传过去：

```bash
# Windows PowerShell 示例（本地执行）
scp -r C:\Users\Caiqing\Desktop\regr root@你的公网IP:/root/
```

然后 SSH 登录服务器：

```bash
cd /root/regr
```

## 第八步：启动项目

仍在**服务器**项目目录执行：

```bash
# 构建并启动（首次会下载镜像，需几分钟）
docker compose up -d --build
```

等待出现 `done` 或类似提示，表示启动完成。

## 第九步：检查服务是否正常

```bash
# 查看容器状态（应看到 mysql、backend、frontend 都是 running）
docker compose ps

# 查看日志（如有报错会在这里）
docker compose logs -f
```

按 `Ctrl+C` 退出日志。没有明显错误即可。

## 第十二步：（可选）配置域名和 HTTPS

### 12.1 域名解析

1. 阿里云控制台 → 域名 → 解析设置
2. 添加 A 记录：主机记录 `@` 或 `www`，记录值填你的**公网 IP**
3. 等待几分钟生效

### 12.2 安装 Nginx 和 Certbot

在**服务器**执行：

```bash
apt install nginx certbot python3-certbot-nginx -y
```

### 12.3 配置 Nginx

```bash
nano /etc/nginx/conf.d/regr.conf
```

写入（把 `你的域名.com` 换成真实域名）：

```nginx
server {
    listen 80;
    server_name 你的域名.com;
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

保存后：

```bash
nginx -t
systemctl reload nginx
```

### 12.4 申请 HTTPS 证书

```bash
certbot --nginx -d 你的域名.com
```

按提示输入邮箱，选择自动重定向 HTTP 到 HTTPS。完成后用 `https://你的域名.com` 访问。

### 12.5 修改 NEXTAUTH_URL

```bash
nano /root/regr/.env
```

把 `NEXTAUTH_URL` 改为 `https://你的域名.com`，保存后：

```bash
cd /root/regr
docker compose restart frontend
```

## 安装为桌面应用（PWA）

项目已支持 PWA，用户可将网站「安装」到桌面，像软件一样使用：

- **电脑**：Chrome / Edge 地址栏会出现安装图标，或访问时底部会提示「安装」
- **手机**：浏览器菜单选择「添加到主屏幕」或「安装应用」
- **iOS**：Safari 分享 → 「添加到主屏幕」

> 需通过 **HTTPS** 访问才能安装（域名 + SSL 证书）。

## 遇到问题？

### Docker 拉取镜像超时 / 失败

国内服务器访问 Docker Hub 较慢，可任选其一：

**方式 A：配置镜像加速器（推荐）**

1. 阿里云控制台 → 容器镜像服务 → [镜像加速器](https://cr.console.aliyun.com/cn-hangzhou/instances/mirrors)
2. 复制你的专属加速地址（如 `https://xxx.mirror.aliyuncs.com`）
3. 编辑 Docker 配置：

```bash
mkdir -p /etc/docker
nano /etc/docker/daemon.json
```

填入（替换为你的加速地址）：

```json
{
  "registry-mirrors": ["https://你的加速地址.mirror.aliyuncs.com"]
}
```

4. 重启 Docker：`systemctl restart docker`

**方式 B：项目已使用 DaoCloud 镜像**

`docker-compose.yml` 中 MySQL 已改为 `docker.m.daocloud.io/library/mysql:8.0`，若仍失败可尝试：

```bash
# 手动拉取
docker pull docker.m.daocloud.io/library/mysql:8.0
```

## 其他问题

- **访问不了**：检查安全组是否放行 80/443/3000
- **容器启动失败**：`docker compose logs` 看具体报错
- **数据库连接失败**：确认 `.env` 里 `MYSQL_ROOT_PASSWORD` 与 `DATABASE_URL` 中密码一致
