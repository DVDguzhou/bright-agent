# 商业计划书（摘要版）

> **BrightAgent** — 个人经验与决策陪伴 · Agent as Service

---

## 执行摘要

- **定位**：在通用大模型难以使用私人/商用数据、建议偏笼统的背景下，提供**可交易、可计量、可存证**的个人经验与决策陪伴服务；以 **Agent** 为载体，由经历者授权其知识与回应方式，用户为**具体场景下的可用答案**付费。
- **核心价值（Agent as Service）**：技术平权后，差异来自**个人数据、社会关系、资产与经验**；平台让个人以 Agent 形式分享并获益。
- **平台职责**：身份与 Agent 认证、License 与短期调用令牌、调用回执与纠纷仲裁；**不托管执行、不转发业务请求**，数据主权在卖方（与产品设计一致）。

---

## 市场痛点

1. 年轻人普遍迷茫，缺少方向感与情绪陪伴。  
2. ChatGPT / Gemini / 豆包等无法安全使用用户私人数据，职业与生活建议**笼统、难落地**。  
3. 难以找到**经历相似**的人；除浏览帖子外，缺少**结构化、可复购**的深度交流机制。

---

## 目标用户

- 在校学生与校招/实习群体  
- 面临职业、生活重大决策的职场人  
- 启动新项目、需要路径与清单的人  
- 到异地/本地旅游、需要**一手经验**的人  

---

## 产品与服务

- **买方**：发现 Life Agent / 场景 Agent，购买使用权，通过对话（含语音）获得可执行建议、清单与避坑信息。  
- **卖方**：创建 Agent、定价与设定能力范围；收益来自 License / 按次或配额计费。  
- **差异化**：InvocationToken + Receipt 对账、可审计调用链；降低「谁说了什么、是否履约」的争议成本。

---

## 商业模式

- 交易抽成（License / 按次）  
- 订阅（买方额度或卖方上架/曝光）  
- B2B2C（机构/社群批量采购）  
- 远期：企业工作流、白标与行业方案  

---

## 竞争与合规要点

- **差异化**：相对公域 LLM 的**深度与授权上下文**；相对社群/星球的可交互、可计量交付；相对纯平台的**数据不经过平台执行层**的合规叙事（需与实现一致）。  
- **风险**：内容真实性、个人信息保护、非医疗非执业咨询的边界说明；用户协议与争议处理机制。

---

## 发展路线（简述）

1. **MVP**：单一垂直场景跑通上架—购买—对话—结算—存证。  
2. **增长**：标签与评价、供给工具、复购与推荐。  
3. **扩展**：多 Agent 编排、机构采购、开发者生态。
