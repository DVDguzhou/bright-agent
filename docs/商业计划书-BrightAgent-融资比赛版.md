# BrightAgent 商业计划书

> 融资 / 比赛展示版  
> 个人经验与决策陪伴 · Agent as Service

---

## 一、执行摘要

BrightAgent 是一个将真实人生经验、社会关系与场景知识封装为可交易 Agent 服务的平台。它面向学生、初入职场者、关键决策阶段用户以及本地生活信息需求者，解决“通用大模型建议笼统、社区内容难以持续追问、真人咨询成本高且难标准化”三类核心问题。

平台的核心思路不是售卖通用 AI 能力，而是让真实经历者将自己的经验、判断方式、关系网络与结构化知识沉淀成可对话 Agent，用户按次、按包或按 License 购买服务。BrightAgent 负责身份认证、交易授权、调用存证、纠纷仲裁与结算信任层；卖方保留 Agent 执行权和私有数据控制权，平台不托管执行、不转发业务请求。

与传统内容社区、通用 LLM、咨询服务相比，BrightAgent 的价值在于三点：第一，提供更具体、更贴近个人处境的决策建议；第二，提供持续对话式的情绪陪伴和追问机制；第三，通过 Token、回执与授权机制，将“经验服务”第一次以标准化、可交易、可审计的方式交付。

本项目当前已具备产品雏形与平台逻辑，包括 Agent 上架、购买 License、签发 `InvocationToken`、校验调用与提交回执等机制。下一阶段将聚焦青年决策场景，优先从“人生经验 Agent / 本地经验 Agent”切入，完成从供给冷启动到复购验证的闭环。

## 二、问题与市场洞察

### 1. 年轻人在关键节点存在高频、高痛感的决策焦虑

无论是升学、实习、求职、转行、创业、异地生活，还是本地消费与短途旅行，年轻人都会遇到两个共同问题：

- 信息很多，但不知道哪一条适合自己
- 需要的不只是信息，还需要“和我情况差不多的人”给出判断

这种需求不是一次性的问答需求，而是包含背景理解、反复追问、风险提醒与情绪安抚的连续决策支持需求。

### 2. 通用大模型擅长总结，但难以提供真实场景下的经验判断

ChatGPT、Gemini、豆包等工具已经极大提升了问答效率，但它们存在天然边界：

- 无法直接、安全、合规地接入大量私人经历与商业私有数据
- 难以知道“你是谁、你处于什么环境、什么方案对你最现实”
- 给出的建议通常是通用方法论，而不是真正可执行的路径建议

这意味着它们适合作为底层推理引擎，但不是最终的“场景化经验服务”产品。

### 3. 现有社区产品解决了浏览，却没有解决高质量交流

小红书、知乎、微博、论坛和群聊解决了“看到别人经验”的问题，但没有解决以下难题：

- 找不到和自己背景真正相似的人
- 帖子是静态的，无法持续追问
- 经验内容分散，缺少结构化交付
- 供给者缺少标准化变现方式，优质经验难以长期供给

因此，市场真正缺少的是一种介于“内容”和“咨询”之间的新型产品：既能低门槛触达，又能形成持续交互和可信交付。

## 三、产品方案

### 1. 产品定义

BrightAgent 是一个经验 Agent 市场和交易信任层平台。用户可以购买并调用由真实经历者创建的 Agent，获得围绕职业、生活、本地经验、项目启动等场景的对话式建议。

目前项目已具备的核心产品表达包括：

- 浏览与创建 `Life Agent`
- 用户进入聊天窗口，与 Agent 持续对话
- 支持按次付费或购买使用权
- 平台对调用进行授权与回执存证

### 2. 核心交付方式

- **经验供给者**：把自己的真实经历、知识库、服务边界和表达风格做成 Agent
- **需求用户**：围绕实际问题发起提问，得到连续、具体、带上下文的建议
- **BrightAgent 平台**：负责认证、授权、计量、存证、交易和纠纷处理

### 3. 为什么这种方案成立

BrightAgent 把原本难以交易的“人生经验”和“本地判断”转化成了可服务化交付的标准单元。它既保留了经验供给的个性化和真实性，又通过平台能力把交付过程标准化、可审计化。

这使得过去难以规模化的一对一经验咨询，第一次可以进入“低客单、高频次、可复购”的消费场景。

## 四、核心价值主张

### 对用户

- 在复杂决策中获得更贴近自身处境的建议
- 不只是看到帖子，而是能持续追问和互动
- 比真人咨询更便宜、更快、更容易获取
- 在需要被理解和陪伴的场景中获得情绪价值

### 对供给者

- 将经验、判断和个人品牌沉淀为可交易资产
- 不需要长期在线，也能通过 Agent 放大服务能力
- 平台提供交易、履约与争议机制，降低信任成本
- 个人数据和私有知识仍由供给者控制

### 对平台

- 形成供给、需求、评价、调用数据与交易记录的复利网络
- 从单次交易向订阅、机构采购和行业方案拓展
- 用“授权 + 存证 + 结算”的信任层，建立中长期壁垒

## 五、目标用户与切入场景

### 1. 核心种子用户

第一阶段，BrightAgent 聚焦最容易形成付费意愿和复购的青年决策场景：

- 在校学生：选专业、选学校、找实习、准备考试、城市选择
- 初入职场者：Offer 选择、转岗、跳槽、行业判断、项目启动
- 新项目发起者：从 0 到 1 的执行路径、避坑清单、资源建议
- 本地生活与旅行用户：本地消费、短途旅游、真实踩坑与推荐

### 2. 初期供给侧画像

- 学长学姐、转行成功者、创业者、城市达人、本地生活意见领袖
- 拥有某一垂直经验优势但没有产品化能力的个人
- 愿意把经验转化为数字服务的内容创作者和社群组织者

### 3. 切入逻辑

BrightAgent 选择的不是“知识最广”的场景，而是“经验差异最有价值”的场景。对这类问题，用户真正购买的不是答案本身，而是“这个人走过这条路之后的判断”。

## 六、市场机会

### 1. 宏观判断

大模型已经完成了用户教育，AI 对话成为日常交互方式；与此同时，越来越多用户发现，通用 AI 并不能替代真实经验判断。这意味着市场机会正从“通用问答”转向“垂直经验服务”。

### 2. 市场结构

BrightAgent 所处的是三个市场的交叉地带：

- AI 应用市场：用户已经习惯通过聊天获得服务
- 内容付费市场：用户愿意为高质量经验与视角付费
- 轻咨询市场：用户需要低于专业咨询门槛、但高于内容浏览的信息服务

### 3. 自上而下的机会判断

以中国学生、初入职场青年和本地生活用户为基础人群，围绕职业决策、人生经验和本地咨询形成的轻咨询市场具备以下特征：

- 场景高频，单价适中
- 用户决策错误成本高，付费意愿强
- 供给端高度分散，但真实经验供给充足

我们认为，BrightAgent 有机会先在垂直场景里建立“经验 Agent 市场”的品类认知，再逐步延伸到更多生活和工作决策场景。

### 4. 市场测算方法

本计划书中的财务模型采用场景驱动测算，而非宽泛 TAM 叙事。核心逻辑是：先验证高频决策类场景中的付费与复购，再向相邻场景复制。

## 七、商业模式

### 1. 当前商业模式

- **按次付费**：用户按问题包、对话包或次数包购买使用权
- **License 授权**：平台面向标准化 Agent 提供可计量调用授权
- **交易抽成**：平台按成交额收取服务费

### 2. 中期商业模式

- **会员订阅**：用户按月购买决策支持额度，提升复购率
- **供给侧增值服务**：上架服务、推荐位、认证标识、运营工具
- **社群 / 机构团购**：学校社团、求职训练营、城市社群批量采购

### 3. 长期商业模式

- **企业场景 Agent 市场**：把经验服务扩展到工作流、行业咨询、专家协作
- **行业解决方案**：围绕教育、出海、电商、营销等行业输出组合型 Agent 方案
- **私有化 / 白标**：向机构提供自有品牌的信任层与调用授权平台

## 八、平台架构与技术壁垒

BrightAgent 的核心不是一个聊天 UI，而是一套可信交易与调用基础设施。

### 1. 平台职责

根据当前项目设计，平台只做四件事：

1. 身份认证  
2. 交易授权  
3. 调用存证  
4. 纠纷仲裁

平台不托管卖方执行服务，不转发真实业务请求。买方持平台签发的短期调用凭证直接调用卖方 Agent，卖方执行后再将回执提交平台。

### 2. 核心机制

- `License`：定义用户购买到的使用权
- `InvocationToken`：定义单次调用的合法凭证
- `ExecutionReceipt`：定义执行结果的回执证明
- 平台对 `request + token + receipt` 进行一致性校验

### 3. 技术壁垒

- 平台不持有核心私有数据，增强供给侧接入意愿
- 交易、存证、对账和争议处理形成标准化基础设施
- 支持从个体 Agent 走向多 Agent 工作流和更复杂的商业场景
- 随着供给增长，评价体系、复购关系和调用记录会形成网络壁垒

技术平权会降低模型能力本身的差异，但不会降低真实经验、供给关系和信任机制的价值。BrightAgent 的壁垒来自“经验资产 + 交易信任层 + 供需网络”。

## 九、竞争分析

### 1. 与通用大模型相比

| 维度 | 通用大模型 | BrightAgent |
|------|------------|-------------|
| 知识来源 | 公域知识为主 | 真实经历者的私有经验与结构化上下文 |
| 交付方式 | 通用问答 | 面向具体场景的经验服务 |
| 建议质量 | 普适性强，个性化弱 | 针对相似背景用户更具体 |
| 商业关系 | 工具订阅 | 供给者变现 + 平台撮合 + 用户付费 |

### 2. 与内容社区相比

| 维度 | 内容社区 | BrightAgent |
|------|----------|-------------|
| 互动方式 | 看帖、评论、私信 | 连续对话、持续追问 |
| 信息组织 | 零散、时间流驱动 | Agent 化、场景化交付 |
| 变现能力 | 广告、电商、知识付费 | 按次咨询、授权、订阅 |
| 供给稳定性 | 高质量作者不稳定 | 可通过 Agent 服务标准化沉淀 |

### 3. 与真人咨询相比

| 维度 | 真人咨询 | BrightAgent |
|------|----------|-------------|
| 成本 | 高 | 中低 |
| 响应速度 | 慢 | 快 |
| 可规模化 | 弱 | 强 |
| 适用场景 | 高客单深度服务 | 中低客单高频决策服务 |

### 4. 竞争结论

BrightAgent 并不与所有咨询或内容平台正面竞争，而是在“真实经验 + 连续对话 + 可交易交付”这个新赛道建立品类。它最有机会成为通用 AI 之上的经验服务层。

## 十、增长策略与运营路径

### 1. 冷启动原则

先做小而深，不做大而泛。优先验证“一个高痛点场景 + 一批高质量供给 + 一套交易闭环”。

### 2. 第一阶段增长路径

- 聚焦 2 到 3 个高需求垂直场景，如校招、转行、本地消费
- 邀请 50 到 100 名种子供给者入驻
- 构建案例内容，在小红书、B 站、知乎等平台建立“真实经验 Agent”认知
- 用首批用户的复购和对话满意度验证模型

### 3. 第二阶段增长路径

- 推出标签、评分、推荐与场景榜单
- 推出供给侧工具：知识导入、定价助手、履约看板
- 与校园组织、训练营、社群、垂类创作者建立合作

### 4. 第三阶段增长路径

- 从 C 端单点场景延伸到 B2B2C 和轻企业服务
- 推出组合型 Agent、专题任务包、多 Agent 工作流
- 从“经验咨询”扩展到“经验驱动的工作流服务”

---

## 十一、阶段目标与里程碑

### 0-6 个月：验证期

- 跑通 Agent 上架、购买、调用、回执与结算闭环
- 完成 3 个垂直场景的需求验证
- 建立首批 50+ 高质量供给者池
- 验证用户付费率、复购率和满意度

### 6-12 个月：增长期

- 建立标签体系与内容分发机制
- 明确最强势的 1 到 2 个核心场景
- 推出订阅与推荐位等增值服务
- 实现 GMV 和供给增长的稳定正循环

### 12-24 个月：扩张期

- 从经验咨询切入机构采购与行业方案
- 将 BrightAgent 从“经验 Agent 市场”升级为“可信 Agent 交易基础设施”
- 打开更多垂直行业与多 Agent 场景

## 十二、财务模型与关键指标

### 1. 模型假设

以下测算为融资 / 比赛展示版的经营模型假设，核心用于展示业务逻辑与增长路径：

- 平台收入由交易抽成、订阅收入和供给侧增值服务组成
- 初期客单价以中低价高频为主，随着场景成熟逐步提升
- 前 12 个月重点验证用户复购与供给稳定性，不以短期利润为首要目标

### 2. 三年预测

| 指标 | 第 1 年 | 第 2 年 | 第 3 年 |
|------|---------|---------|---------|
| 注册用户 | 3 万 | 15 万 | 60 万 |
| 付费用户 | 3,000 | 20,000 | 100,000 |
| 活跃供给者 | 200 | 1,000 | 4,000 |
| 平台 GMV | 300 万元 | 1,800 万元 | 8,000 万元 |
| 平台收入 | 85 万元 | 470 万元 | 2,000 万元 |
| 综合毛利率 | 70%+ | 75%+ | 80%+ |

### 3. 核心指标

BrightAgent 未来最重要的不是 DAU，而是以下指标：

- 付费转化率
- 次月复购率
- 单个 Agent 月均收入
- 优质供给者留存率
- 每单对话完成率与满意度
- 纠纷率与退款率

平台会以“高意图、高付费、高复购”的交易效率指标，替代传统内容平台的流量指标。

### 1. 融资目标

计划融资人民币 300 万至 500 万元，用于完成产品验证、场景突破与供给冷启动。

### 2. 资金用途

### 3. 融资后 12 个月目标

- 建立可持续增长的供给飞轮
- 验证 1 个年 GMV 过千万的核心垂直方向
- 完成平台商业模式验证，并为下一轮融资准备标准化数据包

## 十四、风险与合规

### 1. 内容真实性风险

经验供给本质上来自个人，可能存在夸大、误导或虚假宣传。平台需要通过身份认证、标签审核、评价体系、履约回执和申诉仲裁降低风险。

### 2. 敏感场景边界

对于医疗、法律、金融等高风险行业，BrightAgent 将采用更严格的准入和免责声明机制，避免平台被理解为执业替代服务。

### 3. 数据与隐私

平台坚持“最小必要原则”。核心业务数据尽量保留在供给者侧，平台主要存储交易和回执摘要，以降低隐私泄露与合规压力。

### 4. 供给质量不稳定

平台早期最大的挑战不是需求，而是优质供给的规模化与标准化。BrightAgent 将通过供给工具、模板、激励和标签体系提高供给质量稳定性。

团队若继续推进，需要同时具备三类能力：

- 产品与 AI 应用设计能力
- 平台交易与信任机制建设能力
- 供给冷启动和场景运营能力

## 十六、总结

BrightAgent 所解决的，并不是“AI 能不能回答问题”，而是“在技术平权后，谁能提供真正有价值、可信、可付费的经验服务”。

未来真正稀缺的不是模型本身，而是：

- 个人经历
- 真实关系
- 场景判断
- 可验证的服务能力

BrightAgent 希望成为承载这些稀缺价值的基础平台。它从青年决策与本地经验切入，用 Agent 让每个人的经验第一次以标准化服务的方式被看见、被购买、被复用，并最终形成一个由真实经验驱动的新型交易网络。
