# BrightAgent BP -> Deck 删改清单

> 目标：把 `BP` 里的完整商业叙事，压缩成投资人 `2-3` 分钟可读懂的 `pitch deck`

## 1. 主原则

`BP` 和 `deck` 的写法不同：

- `BP` 追求完整
- `deck` 追求结论先行
- `BP` 可以解释很多背景
- `deck` 必须用最少页面证明“为什么值得继续聊”

因此不要把 `BP` 逐章搬到 `deck`，而要按投资人的注意力顺序重组。

## 2. 可以保留的内容

以下内容可以保留，但要改成更短、更锋利的表达：

### `执行摘要`

保留：

- 真实经历者的经验封装成可交易 Agent
- 平台不是卖通用 AI，而是卖经验服务
- 平台提供认证、授权、存证、结算与纠纷处理

改法：

- 从长段落改成 `1` 句定位 + `3` 个 bullet

### `问题与市场洞察`

保留：

- 年轻人在关键节点有高痛感决策焦虑
- 通用大模型建议泛化
- 内容社区无法持续追问

改法：

- 不要泛泛讲很多人群，只保留 `学生 / 初入职场者`
- 不要把“本地生活”放进主问题页

### `产品方案`

保留：

- 经验供给者 -> Agent -> 用户提问 -> 平台信任层

改法：

- 用一张流程图或 `4` 个步骤表达
- 不要在主 deck 里展开太多协议细节

### `核心价值主张`

保留：

- 对用户更具体、更可持续追问、更低成本
- 对供给者可沉淀为资产、可规模化变现

改法：

- 从“价值口号”改成“为什么用户愿意付费、供给者愿意入驻”

## 3. 需要压缩的内容

### `目标用户与切入场景`

当前问题：

- 用户画像太散，像在同时做多个项目

deck 改法：

- 主叙事只保留 `学生 / 初入职场者的关键决策 Agent`
- `新项目发起者` 和 `本地生活与旅行用户` 放到扩展路径或 appendix

### `市场机会`

当前问题：

- 偏宏观，容易显得空

deck 改法：

- 不讲大而全 TAM
- 改成切入口市场的自下而上估算
- 先证明“这个小市场能成立”，再讲复制路径

### `竞争分析`

当前问题：

- 表格太多，主 deck 容易变重

deck 改法：

- 收敛成 `1` 页
- 只讲一句核心定位：
  `比内容更可追问，比通用 AI 更具体，比真人咨询更低成本`

### `增长策略与运营路径`

当前问题：

- 路线较完整，但还缺与当前阶段匹配的动作感

deck 改法：

- 只保留第一阶段冷启动策略
- 后两阶段缩成 `1-2` 句路线图

## 4. 建议移到 appendix 的内容

这些内容有价值，但不适合主 deck 正文：

- `License / InvocationToken / ExecutionReceipt` 的技术细节
- 完整平台架构说明
- 敏感场景合规边界
- 详细财务预测表
- 长周期扩张路线
- B2B、企业工作流等远期方向

它们适合放在投资人追问时使用，而不是首页面抢叙事。

## 5. 需要新增的内容

这些内容在 BP 中不够强，但 deck 里必须出现：

### `Validation`

必须新增一页，核心内容包括：

- 问卷中 `47/56` 愿意向相似背景前辈求助
- `39/56` 愿意咨询其 AI 化身
- 仅 `13/56` 认为现有 AI 已能满足规划需求
- `34/56` 愿意分享经验训练 AI 并获益

### `Demo / User Story`

必须新增一页具体用户故事：

- 一个学生或初入职场者在关键决策节点如何连续追问
- 为什么这不是一个静态帖子能解决的问题

### `Unit Economics`

必须新增一页或半页：

- 单次付费区间
- 套餐价格带
- 平台抽成
- 模型成本与毛利逻辑

### `Moat`

必须新增一页：

- 不讲“已经形成垄断”
- 讲“随着供给、对话、评分和交易积累，壁垒如何逐步形成”

## 6. 商业模式的改写方向

你在 BP 里写的是：

- 按次付费
- License
- 交易抽成

在 deck 里应改成更容易被投资人追问的版本：

### 早期

- `按次咨询`
- `问题包 / 月度套餐`
- `平台抽成`

### 中期

- 高频用户订阅
- 供给侧工具和推荐位
- 社群 / 训练营 / 校园组织团购

### 后期

- `License`
- 企业化和白标

结论是：  
`先把早期收费模型讲清楚，再保留远期想象，不要同时讲三个阶段的主收入。`

## 7. 关于退出和护城河

### `退出`

不建议在主 deck 单独做一页。  
如果被问到，可以回答：

> 我们按能独立做大的公司来建设，先验证高频决策场景中的经验 Agent 市场；长期如果形成可信 Agent 交易网络，自然会具备战略价值。

### `护城河`

不需要第一天就“已经拥有”，但需要在 deck 中回答：

- 为什么这不是一个容易被复制的聊天应用
- 为什么随着业务增长，壁垒会越来越强

建议统一讲成：

- 真实身份与经历认证
- 经验转 Agent 的工具链
- 连续对话与交易数据
- 标签、评分、推荐与仲裁机制
- 双边网络效应

## 8. 你做 PPT 时可以直接删掉的句型

以下表达在 `BP` 中还能接受，但放到 `deck` 里容易显得空：

- “希望成为……基础平台”
- “处于三个市场交叉地带”
- “最终形成一个新型交易网络”
- “具备广阔市场空间”
- “有机会成为……基础设施”

这些句子不是不能讲，而是要放到最后一页，并且前面必须先有验证和路径支撑。

## 9. 推荐的成稿顺序

你实际做 PPT 时，建议按这个顺序写，而不是按现有 BP 改：

1. 一句话定位
2. 一个高痛点问题
3. 为什么现有替代方案不够
4. 解决方案和示例
5. 验证信号
6. 商业模式
7. 冷启动路径
8. 护城河
9. 市场与扩张
10. 团队与融资用途

## 10. 最终判断

BrightAgent 的想法不是问题，问题在于原 BP 太容易让投资人看到“平台很大”，却看不清“第一枪怎么打、为什么现在能打、凭什么能赢”。

`deck` 的任务不是展示你想得有多全，而是让投资人相信：

- 这是一个真实痛点
- 你找到了一个够窄的切口
- 用户和供给者都已有初步验证
- 这个模式有可能跑出复购和网络效应
